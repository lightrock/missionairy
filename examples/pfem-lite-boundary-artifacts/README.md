# PFEM-lite boundary artifacts

This example pack shows how Doctor Bones turns a small part of PFEM-lite thinking into concrete repo artifacts.

It is not the full PFEM architecture. It is a starter pack for keeping common AI-work boundaries visible:

```text
evidence is not finding
finding is not remediation
remediation is not verification
patch is not proof
completion report is not a vibe
```

## Included example files

- [`completion-report-good.md`](completion-report-good.md)
- [`completion-report-failure.md`](completion-report-failure.md)
- [`completion-report-corrected.md`](completion-report-corrected.md)

## How to use this

Use these examples when reviewing an executor completion note, PR body, or AI answer that claims a coding task is done.

If the answer only says what was changed, compare it to the failure and corrected examples.

If the answer separates changed files, checks run, checks not run, lessons learned, open questions, and verification status, compare it to the good example.
