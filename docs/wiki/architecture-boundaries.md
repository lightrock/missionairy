# Architecture Boundaries

## Core architecture boundary

Humans remain in authority. AI-readable architecture cognition is the operating substrate. Human-facing docs are explanatory/audit views.

## Practical boundaries for assistants

- Do not treat chat memory as durable project memory.
- Do not rewrite doctrine implicitly through implementation drift.
- Do not treat examples or demos as production claims.
- Do not replace bounded workorders with untracked execution chains.

## Folder semantics and cognition

The repo emphasizes semantic folder hierarchy over framework-default folder habits. Folder naming is part of architecture instruction to future humans and AI systems.
